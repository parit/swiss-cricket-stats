Venue View — Final Design

Full spec for the venue feature across the web component

New nav view type

{ type: 'venue', name: 'Bachgraben, Basel' }

Pushed onto the nav stack when a venue name is clicked, or set as root when the venue attribute is present on load.

New component attribute

venue="Bachgraben, Basel"

Loads directly into venue view as root — no back button, consistent with team= and tournament= attributes. Case-insensitive match against ground names in data.json.

Venue layout

Single Matches tab — no Points Table

Past + upcoming matches at that ground, across all tournaments. Tournament pill on every row/card. List/Cards toggle. Back button only when navigated via click (not when set via attribute).

Clickable venue names (Option A — all views)

Ground name becomes a button everywhere a match appears

In _pastTable, _pastCards, _upcomingTable, _upcomingCards — the ground field becomes <button data-cs-venue="Bachgraben, Basel">Bachgraben, Basel</button>, styled like team buttons (dotted underline on hover).
