Key design decisions in this mockup

  • Venue name becomes the view title (ground + city, e.g. Bachgraben, Basel)
  • Only a Matches tab — no Points Table (venues don't have standings)
  • Tournament pills shown on every row (same as team view)
  • List / Cards toggle present
  • Back button shown when navigated via click; absent when venue="…" attribute used
  • Venue names in match rows/cards become clickable buttons (like team names)

One question

Should the venue name also appear in the match row/card in all views (home, tournament, team) as a clickable link — or only in the tournament and team views?

A

All views

Venue is clickable everywhere a match appears — home overview, tournament view, team view

B

Tournament + Team views only

Venue only clickable when already drilled into a specific tournament or team, not on the home overview
